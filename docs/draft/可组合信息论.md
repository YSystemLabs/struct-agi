# 可组合信息论 v0.4

> 副标题：面向 ARC-AGI-3 的交互式信息理解理论
>
> 版本状态：研究纲领 / 理论总纲 / 工程形式化入口
>
> 核心定位：ARC-AGI-2 是最小静态验证场；ARC-AGI-3 是完整动态形态。

---

## 0. 文档定位

本文档提出的“可组合信息论”不是一个已经完成的封闭数学理论，也不是对香农信息论的替代。它更准确地说，是一个面向机器理解、交互式任务求解、结构发现、主动探测、记忆迁移与组合推理的研究纲领。

经典香农信息论解决了信息的度量、压缩、编码、传输与噪声信道恢复问题。但它有意绕开了“意义”问题：信息的语义是什么？机器如何自动理解信息？哪些模式应被提升为对象、变换、组合律、不变量或可复用知识？

可组合信息论试图把这个问题推进一步：

> 机器理解信息，不只是压缩输入或预测标签，而是在交互过程中判断哪些信息值得获取、保留、展开、提升、迁移和重写，并把它们组织成对未来观测、任务解释、动作选择和结构复用近似充分的动态信息态。

这里的“可组合”来自范畴论、Markov 范畴、范畴信息几何、信息瓶颈与结构主义 AI 的共同启发：信息处理不应只被看作孤立映射，而应被看作可以串联、并联、复制、丢弃、条件化、重写、重索引和比较的过程。

这里的“信息论”不是说本文已经给出了类似香农熵那样的唯一基本量，而是说本文试图研究“信息理解”的机械化条件：什么信息能成为结构？什么结构能成为语义？什么语义能跨上下文、跨任务、跨时间尺度复用？

本版本 v0.4 相比 v0.3 的最大变化是：

1. 不再把 ARC-AGI-2 作为理论主场，而是把它定位为最小静态验证场；
2. 把 ARC-AGI-3 作为理论终极形态；
3. 从静态结构世界升级为交互式动态信息态；
4. 把任务信念、历史压缩、主动探测、动作价值、惰性展开、分层记忆、重索引和反误归纳纳入理论主体；
5. 把“结构发现理论”提升为“交互式信息理解理论”。

---

## 1. 一句话定义

可组合信息论研究的是：

> 在目标语义不显式、观测历史不断增长、动作具有成本与信息价值、结构解释需要跨语境迁移的交互式任务中，agent 如何把原始观测压缩为局部最小充分的信息状态、任务信念、结构候选和分层记忆，并通过发现、筛选、提升、重写、重索引形成可组合、可干预、可迁移的语义结构。

更短地说：

> 理解一段信息，就是在交互中判断它是否值得获取、保留、展开、提升、迁移和重写，并将其压缩进对未来观测、任务解释、动作选择和结构复用近似充分的动态信息态中。

## 2. 命名动机：为什么叫“可组合信息论”

### 2.1 为什么仍叫“信息论”

经典信息论回答的是：

* 一个信源有多少不确定性；
* 一段消息有多少信息量；
* 信息如何压缩；
* 信息如何编码；
* 信息如何传输；
* 噪声下如何恢复信号；
* 信道容量和编码极限是什么。

它没有正面回答：

* 信息的语义是什么；
* 机器如何自动理解信息；
* 对象、关系、变换、不变量从何而来；
* 哪些模式值得被提升为语义；
* 为什么某些结构能迁移，某些结构只是伪规律；
* 一个系统如何在交互中决定什么信息值得获取。

因此，可组合信息论不是否定香农信息论，而是把香农主动悬置的“意义问题”重新放入可机械化研究的范围。

它试图回答：

> 如果信息不仅要被传输，还要被机器理解，那么什么才是有理解价值的信息？

### 2.2 为什么叫“可组合”

“可组合”来自三个方面。

第一，来自范畴论的启发。对象、态射、复合、恒等、函子、纤维化、重索引等语言，天然适合描述结构如何跨上下文保持关系。

第二，来自 Markov 范畴和范畴信息几何的启发。信息处理过程可以被看作可组合 morphism；copy、discard、composition、divergence、data processing 等概念可以被放入统一过程语言中。

第三，来自 ARC-AGI 类任务本身。ARC 任务的关键不是大规模拟合，而是少样本下发现对象、关系、变换、组合规律和不变量。一个结构如果不能组合、不能迁移、不能重写，就很难称为真正语义。

因此，“可组合”不是修饰词，而是理论核心：

> 信息只有在组合、干预、迁移和重写中保持稳定结构角色，才有资格成为语义信息。

## 3. ARC-AGI-2 与 ARC-AGI-3 的关系

### 3.1 ARC-AGI-2：最小静态验证场

ARC-AGI-2 可以被看作可组合信息论的最小静态切片。

它的问题形态是：

```text
有限 train pairs → 结构发现 → 最小充分解释 → test prediction
```

在 ARC2 中，系统主要验证以下能力：

* 能否从网格中发现对象候选；
* 能否显式表示关系；
* 能否发现输入输出之间的变换；
* 能否筛选最小充分解释；
* 能否用 MDL、残差、扰动稳定性过滤伪结构；
* 能否通过 structure-miner 复用成功解释；
* 能否通过重写避免知识堆积。

因此，ARC2 验证的是：

> 静态观测中，能否从有限样例发现可组合结构。

但 ARC2 不验证完整智能体问题。因为 ARC2 中：

* 观测已经给定；
* train pairs 已经提供；
* 动作不改变未来观测；
* 没有显式动作成本；
* 没有长期历史增长；
* 不需要主动探测；
* 不需要跨 level 机制迁移；
* 不需要持续维护任务信念。

所以 ARC2 是必要验证场，但不是充分验证场。

### 3.2 ARC-AGI-3：完整动态形态

ARC-AGI-3 更接近可组合信息论真正要处理的问题。

它的问题形态是：

```text
历史观测 → 动态信息态 → 动作选择 → 环境反馈 → 信息态更新
```

或者写成闭环：

```text
β_t → a_t → o_{t+1} → β_{t+1}
```

其中：

```text
β_t = (i_t, B_t, X_t, M_t)
```

* i_t：信息状态，负责压缩历史；
* B_t：任务信念，负责表示当前可能目标和解释后验；
* X_t：结构候选池，负责维护对象、关系、机制、变换和模型假设；
* M_t：分层记忆，负责工作记忆、同游戏机制记忆和长期知识。

ARC-AGI-3 逼出的问题是：

* 当前历史中什么信息值得保留？
* 当前任务目标到底是什么？
* 哪些结构解释仍然存活？
* 哪些候选值得展开？
* 哪些动作值得执行？
* 哪些动作只是为了获取信息？
* 哪些局部规律能提升为 game-level mechanism？
* 哪些机制能进入长期知识？
* 旧结构在新语境下如何重解释？
* 如何避免误归纳？

因此，ARC-AGI-3 验证的是：

> 交互过程中，agent 能否主动管理信息流，并把信息转化为可组合、可迁移、可重写的结构能力。

### 3.3 两者关系

ARC2 和 ARC3 的关系不是替代关系，而是层级关系：

```text
ARC-AGI-2：静态结构发现子问题
ARC-AGI-3：动态信息理解完整问题
```

ARC2 中验证的对象发现、结构筛选、残差归因、MDL、重写、structure-miner，都是 ARC3 中动态智能体的子模块。

因此，合理路线是：

1. 用 ARC2 验证最小静态结构发现；
2. 用 ARC2 的失败归因发展结构增长与重写；
3. 把这些能力嵌入 ARC3 的动态信息态；
4. 在 ARC3 中验证主动探测、任务信念、分层记忆和跨语境迁移。

## 4. 从静态结构世界到动态信息态

v0.3 主要使用静态结构世界：

```text
L = (O, M, C, I, R)
```

其中：

* O：对象类；
* M：态射 / 变换类；
* C：组合规则；
* I：不变量与约束；
* R：重写与等价规则。

这个结构对 ARC2 合适，但对 ARC3 不够。因为 ARC3 中系统不只是发现结构，还必须在交互中管理信息。

因此 v0.4 将核心对象升级为动态信息态：

```text
β_t = (i_t, B_t, X_t, M_t)
```

如果把控制与约束也显式展开，可以写成：

```text
A_t = (i_t, B_t, X_t, M_t, Π_t, C_t)
```

其中：

* i_t：历史压缩后的信息状态；
* B_t：任务信念对象；
* X_t：结构候选池；
* M_t：分层记忆；
* Π_t：当前策略 / 控制前沿；
* C_t：预算、风险、可逆性和动作约束。

这不是随意加模块，而是 ARC3 的接口压力自然推出的分解。

因为单一 latent state 很难同时服务：

* 下一步观测预测；
* 当前任务目标判别；
* 结构机制发现；
* 动作价值排序；
* 探测动作选择；
* 记忆迁移；
* 知识晋升；
* 反误归纳。

所以可组合信息论的动态版本主张：

> 智能体不是寻找一个万能表示，而是在不同职责边界上构造多个局部最小充分表示，并通过可组合信息通道连接它们。

## 5. 分层局部信息瓶颈

信息瓶颈的基本思想是：表示应尽量压缩输入，同时保留对目标变量有用的信息。

普通形式可以理解为：

```text
X → Z → Y
```

其中 Z 压缩 X，但保留关于 Y 的信息。

在 ARC3 中，问题更复杂。因为不同层的“目标变量”不同。

信息状态层、任务信念层、结构候选层、控制层、记忆层不应该压缩成同一个 Z。它们各自服务不同的下游变量。

### 5.1 信息状态层的局部瓶颈

信息状态 i_t 的输入是历史：

```text
H_t = (o_0, a_0, o_1, a_1, ..., o_t)
```

但 i_t 不应保存全部历史。它应保存对以下内容近似充分的信息：

* 下一步观测摘要；
* 当前目标推进摘要；
* 当前动作排序所需 sufficient statistics；
* 结构层和任务层所需的关键历史证据。

因此，i_t 的任务不是记住一切，而是压缩历史、丢弃冗余、重组证据、暴露残差。

### 5.2 任务信念层的局部瓶颈

任务信念 B_t 的目标不是预测全部环境，而是区分当前可能的任务解释。

它要保留的是：

* 哪些目标解释仍然可能；
* 哪些观测支持或反驳某个解释；
* 哪些动作能最大区分竞争任务；
* 哪些进展信号说明目标接近完成。

任务信念层的核心信息论问题是：

> 什么观测最能减少任务解释后验的不确定性？

### 5.3 结构候选层的局部瓶颈

结构候选 X_t 保存的不是所有可能结构，而是在当前任务信念下仍有解释价值的候选。

它要保留的是：

* 对象候选；
* 关系候选；
* 环境机制候选；
* 可执行变换候选；
* 因果/可逆性假设；
* 对未来观测和动作反馈有判别力的模型。

结构候选层的核心信息论问题是：

> 哪些结构候选能最大压缩未来观测和动作反馈的不确定性？

### 5.4 控制层的局部瓶颈

控制层不是尾部执行器，而是内部信息流和外部环境通道的接口。

它需要压缩成动作排序所需信息：

* 动作能推进目标吗？
* 动作能区分任务解释吗？
* 动作能区分结构模型吗？
* 动作是否可逆？
* 动作失败代价多大？
* 动作会打开还是关闭未来信息通道？

控制层的核心信息论问题是：

> 哪些动作最值得用成本换取进展或信息？

### 5.5 记忆层的局部瓶颈

记忆层不应保存所有经验，而应保存跨时间尺度可复用的结构。

它分为：

* 工作记忆：当前 level 或当前局部阶段所需；
* game-level 机制记忆：同一游戏内多个 level 复用；
* 长期知识：跨游戏、跨任务族复用。

记忆层的核心信息论问题是：

> 哪些局部模式值得晋升为更长时间尺度上的机制？

### 5.6 分层瓶颈原则

因此，可组合信息论中的信息瓶颈不是单一瓶颈，而是分层局部瓶颈：

```text
每一层都压缩上一层输入，保留对本层职责相关的下游变量近似充分的信息。
```

这解释了为什么不能用一个单一 world model 或单一 latent state 代替整个系统。

## 6. 信息流闭环

ARC3 中的 agent 信息流可以写成：

```text
H_t → i_t → (B_t, X_t, M_t) → Π_t → a_t → o_{t+1} → update → β_{t+1}
```

每一步都不是简单函数拟合，而是一次可诊断的信息变换。

### 6.1 历史到信息状态

```text
H_t → i_t
```

作用：

* 压缩历史；
* 删除冗余；
* 重组证据；
* 暴露状态变量；
* 保留动作反馈；
* 标记未解释事件。

失败形式：

* 忘掉关键证据；
* 保留过多噪声；
* 混淆不同阶段；
* 无法恢复动作后果；
* 无法支持任务信念更新。

### 6.2 信息状态到任务信念

```text
i_t → B_t
```

作用：

* 维护竞争目标解释；
* 根据新观测更新后验；
* 判断哪些解释仍存活；
* 识别任务进展信号。

失败形式：

* 过早锁定错误目标；
* 不保留多解释；
* 无法利用反例；
* 无法区分目标进展和局部变化。

### 6.3 信息状态到结构候选

```text
i_t → X_t
```

作用：

* 产生对象候选；
* 产生关系候选；
* 产生机制候选；
* 产生环境模型；
* 维护候选后验。

失败形式：

* 正确对象不存在；
* 正确关系未显式化；
* 机制候选缺失；
* 候选过多导致搜索爆炸；
* 候选无法解释动作反馈。

### 6.4 信念与结构到控制

```text
(B_t, X_t, M_t, C_t) → Π_t → a_t
```

作用：

* 对动作排序；
* 平衡探索与利用；
* 考虑动作成本；
* 考虑风险和可逆性；
* 考虑信息增益；
* 考虑长期机制复用。

失败形式：

* 执行了不可逆错误动作；
* 探测不足；
* 过度探测；
* 忽略低成本信息动作；
* 无法区分进展动作和探测动作。

### 6.5 反馈到更新

```text
(a_t, o_{t+1}) → β_{t+1}
```

作用：

* 更新信息状态；
* 更新任务信念；
* 更新结构候选；
* 更新记忆；
* 记录支持集和反例集；
* 触发结构重写或晋升。

失败形式：

* 不吸收反例；
* 错误晋升；
* 无法撤销错误信念；
* 无法利用反馈压缩候选集合。

## 7. 可组合信息的生命周期

在 ARC3 中，信息不是一次性被抽取，而是经历生命周期。

```text
发现 → 筛选 → 展开 → 提升 → 重写 → 重索引 → 迁移 / 冻结 / 淘汰
```

### 7.1 发现

发现是从观测历史中产生候选结构。

候选可以是：

* 对象候选；
* 关系候选；
* 任务目标候选；
* 机制候选；
* 动作效果候选；
* 策略骨架候选；
* 记忆候选。

发现不等于理解。发现只是把可能有用的信息从背景中凸显出来。

### 7.2 筛选

筛选是用证据压缩候选后验。

筛选依据包括：

* 解释当前观测；
* 预测未来反馈；
* 区分竞争解释；
* 降低残差；
* 降低描述长度；
* 提高扰动稳定性；
* 提高组合稳定性。

### 7.3 展开

展开是把潜在候选显式计算出来。

ARC3 中不能把所有候选都展开，因为计算预算有限。因此需要惰性机制。

一个候选是否值得展开，取决于：

* 它能否区分任务信念；
* 它能否区分结构模型；
* 它能否改善动作排序；
* 它是否有复用潜力；
* 它的展开成本多高。

### 7.4 提升

提升是把局部有效结构晋升到更高层级。

例如：

```text
局部模式 → level structure → game mechanism → long-term knowledge
```

提升必须满足：

* 有支持集；
* 有反例集；
* 有适用条件；
* 有置信度；
* 有寿命或衰减机制；
* 通过 no-regression gate。

### 7.5 重写

重写是把多个表面不同但行为等价或近似等价的结构折叠。

重写包括：

* canonicalization；
* alias elimination；
* macro folding；
* macro unfolding；
* dead knowledge freezing；
* equivalence class compression。

没有重写，知识增长会退化成规则堆积。

### 7.6 重索引

重索引是把旧结构放到新任务语境中重新解释。

ARC3 中，任务语境会不断变化：

* 目标解释改变；
* level 阶段改变；
* 物体角色改变；
* 动作后环境结构改变；
* game mechanism 逐渐显露。

因此，旧结构不应简单丢弃，而应问：

> 在新语境下，它还表示同一个角色吗？还是需要换一种解释？

这就是纤维化视角的核心价值：结构不是脱离语境的静态对象，而是在任务语境上方的候选族；语境变化时，结构需要重索引。

---

## 8. 惰性展开：候选、展开、执行三者分离

ARC3 中必须区分三件事：

```text
Candidate ≠ Expanded Structure ≠ Executed Action
```

### 8.1 Candidate

候选只是潜在解释。

它可能表示：

* 一个对象划分；
* 一个目标解释；
* 一个动作效果模型；
* 一个可疑机制；
* 一个策略骨架。

候选不需要立即完全展开。

### 8.2 Expanded Structure

展开结构是被显式计算、比较、评分后的候选。

展开会消耗内部计算预算。

展开的价值在于：

* 减少任务信念熵；
* 减少结构后验熵；
* 改善动作排序；
* 暴露可复用机制；
* 发现反例。

### 8.3 Executed Action

执行动作是 agent 与环境交互。

它会消耗外部动作预算，可能带来风险，也可能不可逆。

因此：

> 一个候选值得展开，不等于对应动作值得执行。

惰性展开是内部信息计算策略；动作选择是外部信息采样与控制策略。二者相关，但不能混为一谈。

---

## 9. 控制作为信息采样

在 ARC3 中，动作不是单纯完成目标的手段，也可能是获取信息的手段。

一个动作的价值应同时考虑：

```text
ActionValue(a)
= progress value
+ information gain
+ model discrimination value
+ reversibility value
- action cost
- risk
- opportunity cost
```

这不是最终理论公式，而是工程代理表达。

### 9.1 进展价值

动作是否让任务目标更接近完成？

### 9.2 信息增益

动作是否能减少任务信念不确定性？

例如，同一个按钮可能揭示环境机制，哪怕它暂时不推进目标。

### 9.3 模型区分价值

动作是否能区分两个竞争结构模型？

例如，两个候选机制对当前状态预测相同，但对某个探测动作预测不同。

### 9.4 可逆性价值

动作是否可撤销？

在不确定阶段，可逆动作通常更适合探测。

### 9.5 风险与机会成本

动作是否会关闭未来路径？是否会破坏关键对象？是否会使某些信息永远不可观测？

因此，控制层是可组合信息论的一部分，因为它决定了 agent 如何主动采样信息。

---

## 10. 语义：被协议筛选后的结构角色

可组合信息论中的“语义”不是人类给的标签，也不是模型内部神秘向量。

语义是一种操作性地位：

> 一个结构单元具有语义，当且仅当它在任务协议、组合协议、干预协议、迁移协议和重写协议下持续保持可解释、可预测、可复用的角色。

### 10.1 合法语义

合法语义通常满足：

* 降低任务不确定性；
* 降低描述长度；
* 经受允许扰动；
* 支持组合外推；
* 能用于动作选择；
* 能跨 level 或跨任务复用；
* 能被重写系统规范化；
* 有支持集和反例集。

### 10.2 伪语义

伪语义通常表现为：

* 只在少数样例中偶然成立；
* 依赖绝对颜色编号；
* 依赖绝对坐标；
* 轻微扰动即失效；
* 不能支持组合；
* 无法指导动作；
* 跨 level 失效；
* 增加知识库复杂度但不带来复用收益。

因此：

> 语义不是稳定模式本身；语义是经过任务、组合、干预、迁移、动作反馈和重写检验后仍保持角色的结构信息。

---

## 11. 时间尺度与反误归纳

ARC3 中信息的价值和时间尺度强相关。

一个模式在当前局部阶段有效，不代表它能成为长期知识。

### 11.1 四级晋升链

```text
local observation
→ level structure
→ game mechanism
→ long-term knowledge
```

### 11.2 local observation

局部观测只说明当前状态中发生了某件事。

例如：某个对象碰到墙后改变颜色。

它还不是机制。

### 11.3 level structure

如果一个模式能解释当前 level 中多次反馈，它可以成为 level structure。

但它仍可能只是当前 level 特例。

### 11.4 game mechanism

如果一个结构在同一个 game 的多个 level 或多个阶段中复用，它可以成为 game-level mechanism。

例如：某类对象总是钥匙，某种颜色总是危险区域，某个操作总是打开通道。

### 11.5 long-term knowledge

如果一个结构跨 game、跨任务族复用，且有清晰适用条件和反例边界，它才有资格进入长期知识。

### 11.6 反误归纳原则

每次晋升都必须记录：

* support set；
* counterexample set；
* applicability condition；
* confidence；
* lifetime；
* decay rule；
* no-regression test。

反误归纳不是安全附录，而是语义晋升的守门机制。

---

## 12. 结构图式失配

结构图式是系统当前用来解释世界的对象、关系、机制、动作效果、组合规则和约束类型。

结构图式失配指：

> 环境中的真实任务结构，无法被当前候选结构世界以低失真、低残差、低描述长度、低风险的方式表达。

在 ARC3 中，结构图式失配可能表现为：

* perception fail：正确对象没有出现；
* relation fail：对象存在，但关键关系未显式化；
* mechanism fail：动作效果模型错误；
* task fail：目标解释错误；
* composition fail：局部机制无法组合；
* control fail：正确结构存在，但动作排序错误；
* memory fail：旧机制误迁移；
* reindex fail：旧结构在新语境下没有正确重解释；
* promotion fail：局部规律被错误晋升；
* rewrite fail：等价结构没有折叠，导致搜索爆炸。

因此，失败不是简单的“输出错了”，而是某个信息处理通道没有完成其结构压缩职责。

---

## 13. ARC2 子系统如何服务 ARC3

ARC2 中的每个模块都应被看成 ARC3 动态智能体的静态子系统验证。

| ARC2 模块                  | 在 ARC2 中验证什么                       | 在 ARC3 中对应什么           |
| ------------------------ | ---------------------------------- | ---------------------- |
| 多预序感知层                   | 从网格产生前语义关系场                        | 信息状态中的局部关系显式化          |
| proto-object discovery   | 产生对象候选                             | 结构候选池 X_t 的对象部分        |
| object graph             | 显式关系与属性                            | 当前环境结构假设               |
| candidate transform      | 输入输出结构差分                           | 动作效果 / 机制候选            |
| MDL / residual           | 筛选最小解释                             | 任务信念与结构信念更新            |
| perturbation consistency | 过滤伪结构                              | 干预稳定性检验                |
| structure-miner          | 离线发现 selector / constraint / macro | game mechanism 和长期知识晋升 |
| rewrite                  | 防止规则堆积                             | 分层记忆压缩与知识规范化           |
| no-regression            | 防止污染已解任务                           | 反误归纳 gate              |

所以 ARC2 的价值不是“解 ARC2 本身”，而是验证 ARC3 所需的基础结构能力是否可工程化。

## 15. 最小研究路线

### Phase A：ARC2 静态结构验证

目标：证明前语义关系层、对象发现、结构筛选和重写不是空想。

关键实验：

* multi-preorder minimal validation；
* 固定模板池下对象候选比较；
* MDL + residual + perturbation consistency；
* structure-miner 离线候选发现；
* no-regression gate。

### Phase B：ARC2 到 ARC3 的桥接

目标：把 ARC2 静态结构解释转化为 ARC3 中可维护的结构候选和机制候选。

关键问题：

* 静态对象候选如何进入 X_t；
* 静态变换如何变成动作效果模型；
* 静态 residual 如何变成探测需求；
* 静态 macro 如何变成策略骨架；
* 静态重写如何变成记忆压缩。

### Phase C：ARC3 动态信息态

目标：实现 β_t = (i_t, B_t, X_t, M_t) 的最小闭环。

最小能力：

* 历史摘要；
* 多任务信念；
* 结构候选维护；
* 动作后更新；
* 探测动作选择；
* 简单机制记忆。

### Phase D：惰性展开与主动探测

目标：区分 candidate、expanded structure、executed action。

关键实验：

* 展开预算受限；
* 探测动作有成本；
* competing hypothesis 需要主动区分；
* 比较 eager vs lazy；
* 比较 random probe vs information-directed probe。

### Phase E：机制晋升与反误归纳

目标：验证 local → level → game → long-term 的晋升纪律。

关键实验：

* 同 game 多 level 复用；
* support/counterexample 记录；
* false promotion 检测；
* memory decay；
* no-regression。

### Phase F：重索引与跨语境迁移

目标：验证旧结构能否在新任务语境中被重新解释，而不是简单复用或丢弃。

关键实验：

* 角色变化；
* 目标变化；
* 相同对象不同功能；
* 同一机制在不同 level 下不同表现；
* reindex 成功率。

---

## 16. 与其他路线的区别

### 16.1 不同于普通 world model

普通 world model 倾向于学习一个统一环境状态和转移预测。

可组合信息论认为：单一 world model 不足以承担所有信息职责。agent 需要分层局部瓶颈：历史压缩、任务信念、结构候选、控制前沿、分层记忆各有不同保真目标。

### 16.2 不同于普通强化学习

普通强化学习通常通过 reward 优化策略。

可组合信息论强调：在目标语义不显式时，agent 首先要管理任务信念和结构候选；动作不仅有 reward/progress 价值，还有信息价值、模型区分价值和可逆性价值。

### 16.3 不同于传统符号 AI

传统符号 AI 往往默认符号和语义已经给定。

可组合信息论关心的是：符号和语义如何从观测、动作反馈和跨时间尺度复用中被筛选、晋升和重写出来。

### 16.4 不同于普通程序归纳

程序归纳问：哪个程序解释样例？

可组合信息论进一步问：这个程序暴露了什么结构？这些结构能否复用？能否指导动作？能否跨 level 迁移？能否被重写系统压缩？

### 16.5 不同于单纯 MDL

MDL 关注短描述。

可组合信息论关注可组合的短描述：短规则只有在组合、干预、迁移、动作反馈和重写中保持角色，才是语义结构。

---

## 17. 理论边界与反误读

### 17.1 不是香农信息论的替代

可组合信息论不是推翻香农信息论，而是试图研究香农信息论有意不处理的理解问题。

### 17.2 不是完成的公理化理论

v0.4 是研究纲领和工程形式化入口，不声称给出完备公理系、唯一信息量或一般表示定理。

### 17.3 不是“范畴论解释一切”

范畴论在这里首先是组合、重索引、纤维化和结构保持的组织语言，不是要求系统一开始输出完整范畴。

### 17.4 不是 ARC2 solver 理论

ARC2 是静态子问题，ARC3 才是理论完整目标。

### 17.5 不是所有稳定模式都是语义

稳定模式只有通过任务、组合、干预、迁移、动作反馈、时间尺度和重写检验，才有资格晋升为语义。

### 17.6 不是越复杂越好

动态信息态的拆分不是为了堆模块，而是因为不同层有不同信息保真目标。任何模块如果不能对应明确的信息职责，都应删除或合并。

---

## 18. 核心判断

可组合信息论的合理定位是：

```text
ARC-AGI-2：验证静态结构发现是否成立。
ARC-AGI-3：验证交互式信息理解是否成立。
```

ARC2 让理论避免空谈，因为它要求对象发现、结构筛选、残差归因、重写和复用都能落到有限任务上。

ARC3 让理论真正完整，因为它引入了历史、动作、目标不显式、探测、风险、记忆、迁移和重索引。

因此，v0.4 的最终主张是：

> 可组合信息论不是为静态 ARC solver 服务的理论，而是为 ARC-AGI-3 这种交互式、目标不显式、动作有信息价值、历史需压缩、结构需迁移的智能体建立的底层信息观。ARC-AGI-2 是它的最小静态切片。

---

## 19. 最终压缩版

如果只用一段话概括：

> 可组合信息论试图把“机器如何理解信息”机械化。经典信息论研究信息如何度量、压缩和传输；信息瓶颈研究表示如何压缩输入并保留任务相关信息；可组合信息论进一步研究，在交互式任务中，agent 如何判断哪些信息值得获取、保留、展开、提升、迁移和重写，并将其组织成对未来观测、任务解释、动作选择和结构复用近似充分的动态信息态。ARC-AGI-2 提供静态结构发现的最小验证场；ARC-AGI-3 提供完整的动态信息理解场景。真正的语义不是稳定模式本身，而是在任务、组合、干预、迁移、动作反馈和重写协议下持续保持角色的结构信息。
